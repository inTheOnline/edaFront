# base/routeView 路由可视化管理页

## 阅读顺序
- 修改本目录代码前，先读本 README.md。
- 当前页面如果继续拆分子组件、hooks 或独立服务文件，修改前先一起读。

## 功能内容
- 用于可视化新增、查看、编辑、删除动态路由。
- 页面采用左树右详情布局，左右面板保持同高；左侧专注拓扑编排，右侧专注当前选中节点的只读详情。
- 支持拖拽调整目录挂载关系，例如把 A 下的 A1 拖到 B 下，直接变成 B 的子模块。
- 支持顶级路由排序、同一父级下兄弟节点排序、跨目录拖拽变更父子关系。
- 支持配置目录路由、页面路由、父级模块、标题、路径、组件路径、权限标识、缓存/隐藏/全屏等 meta 信息。
- 新增路由通过双栏弹窗完成，左边表单、右边请求预览；编辑、删除都先二次确认，再进入弹窗或调用接口。
- 支持直接选择菜单 icon，当前对接项目现有的 Element Plus 图标体系。
- 支持在新增/编辑弹窗内实时预览 payload，便于联调后端菜单表接口。
- 详情面板除基础字段外，还补充 `meta` 聚焦展示区和“路由说明/操作提示”区，避免右侧过短。

## 技术实现
- 页面优先请求约定接口 `GET /menu/tree` 获取当前路由树。
- 当后端接口还没实现时，会回退到当前登录用户菜单或本地 `authMenuList.json` 做展示，不影响页面使用。
- 保存时按约定接口 `POST /menu/add` 提交。
- 编辑时按约定接口 `POST /menu/update` 提交。
- 拖拽挂载时按约定接口 `POST /menu/move` 提交移动信息。
- 删除时按约定接口 `POST /menu/delete` 提交。
- icon 选择复用全局组件 `SelectIcon`，底层使用 `@element-plus/icons-vue`。
- 路由拓扑区改为紧凑型递归拖拽树，拖拽同级排序时显示水平插入线，拖到目录节点上时高亮父节点并提示“将成为 XX 的子节点”。
- 为了避免页面过长，新增/编辑表单都放进独立弹窗，主页面只保留高频查看区和当前节点详情。

## 代码习惯规范
- 该页是基础管理下的新模块，只在本目录内维护，不改动旧模块。
- 提交给后端的 `component` 路径保持和现有动态路由一致，不带 `.vue` 后缀，例如 `/base/routeView/index`。
- 父级目录型路由允许没有 `component` 和 `name`，页面型路由需要填写。
- 当前拖拽只允许挂到目录路由下，不允许把节点挂到页面路由下。
- 默认只展开顶级目录，避免树过长影响操作；需要时再手动展开或点“展开全部”。
