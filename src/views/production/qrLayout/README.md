# production/qrLayout 二维码标签设计器

## 阅读顺序
- 修改本目录代码前，先读本 README.md。
- 主要实现位于 `src/components/qrLayout`，本页面只做生产系统入口承载。
- 当前需求要求不改其他模块；路由接入需要由后端动态菜单配置或单独确认后再修改现有路由文件。

## 功能内容
- 企业级二维码/条码标签模板编辑工具。
- 支持二维码、Code128、文本、图片、线条、矩形混排。
- 支持对象树、对象级数据源管理、流水号批量规则、模板示例库、JSON 模板导入导出。
- 支持网格吸附、第三方 Moveable 交互、拖拽、缩放、旋转、复制、删除、锁定、隐藏、层级调整和精确坐标录入。
- 文本对象支持双击编辑，选中对象支持 Delete/Backspace 删除。
- 二维码和 Code128 的扫描内容文字作为对象附属属性处理，可显示/隐藏，并配置相对位置和 X/Y 偏移。
- 支持滚轮缩放画布，缩放范围 10% 到 800%；Shift 缩放保持比例，二维码和图片默认保持比例。
- 支持浏览器预览打印、测试打印、TSPL2 指令生成和本地打印服务调用入口。

## 技术实现
- 页面使用独立全屏工具形态，不走 ProTable。
- 核心组件：`src/components/qrLayout/index.vue`。
- 类型定义：`src/components/qrLayout/types.ts`。
- 示例模板：`src/components/qrLayout/templates.ts`。
- 工具方法：`src/components/qrLayout/utils.ts`。
- 二维码生成复用项目已有 `qrcode` 依赖。
- 画布选中、拖拽、缩放、旋转试用 `vue3-moveable`，如果后续体验不符合要求，需要同步删除依赖和组件引用。
- Code128 预览使用前端编码绘制，TSPL 打印使用打印机原生 `BARCODE 128` 指令。
- 得力 DL-720C 按 203dpi 处理，TSPL 坐标换算约为 `1mm = 8 dots`。
- 当前数据源模型跟随对象保存：每个文本、二维码、Code128 对象都有自己的表达式和数据源列表。
- 数据源支持文本/数值，来源支持固定值、打印输入、日期、时间、流水号、拼接、公式，并支持前缀、后缀、补位、替换、大小写、空值默认等转换。

## 后端 API
- 当前无后端 API。
- 模板保存为前端 JSON 文件，图片允许 base64 内嵌。
- 直连打印预留本地服务：`POST http://127.0.0.1:17620/print`，请求体包含 `tspl` 和 `template`。

## 菜单/路由接入
- 动态菜单 component 建议配置为 `/production/qrLayout/index`。
- path 建议配置为 `/production/qrLayout`。
- title 建议为 `二维码标签设计器`。
- 如果需要前端静态路由兜底，必须先获得用户确认后再修改 `src/routers`。

## 注意点
- 当前不做 WebUSB/WebSerial。
- 当前不做 Excel/CSV 导入。
- 本地 Node 打印服务放在前端设计器稳定后再实现。
- TSPL 图片输出当前先生成占位注释，后续本地服务阶段再统一把图片转黑白位图写入打印机指令。
- 页面高度按管理系统内容区处理，避免工具页自身再撑出整页滚动；中间画布区域保留类似 PDF/PS 工具的滚动条。
- 打印弹窗会收集各对象中类型为打印输入和流水号的数据源，其他数据源按对象自身配置自动生成。
